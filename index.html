1. Data Handling
Definition

Data handling is the process of ensuring that research data is stored, archived, or disposed of in a safe and secure manner during and after the conclusion of a research project.

It includes collecting, organizing, storing, protecting, and processing data so it can be accessed and used correctly.

2. Data Wrangling
Definition

Data Wrangling (also called Data Munging) is the process of gathering, collecting, and transforming raw data into another format for better understanding, decision-making, accessing, and analysis in less time.

It involves processing data in various formats like merging, grouping, concatenating, etc. to prepare it for analysis.

Data Wrangling Process — 5 Steps

  1. Data Acquisition — Obtain data from various sources (databases, APIs, CSV files) and store in a central location.
  2. Data Cleaning — Fix errors, remove duplicates, handle missing values to ensure accuracy and consistency.
  3. Data Exploration — Explore the data to understand its structure, content, and relationships.
  4. Data Transformation — Convert data into a format more appropriate for analysis (normalize, encode, reshape).
  5. Data Loading — Load the prepared data into the analysis tool or platform (pandas, SQL, ML model).

4 Core Operations in Data Wrangling

CLEAN

Cleaning ensures data is accurate and consistent. It includes:

1. Data Inspection — Review data to find errors, inconsistencies, missing values.

2. Data Validation — Check data against rules/constraints to verify it is correct.

3. Data Correction — Fix errors: fill missing values, remove duplicates, correct wrong entries.

4. Data Standardization — Ensure consistent format (e.g., all dates in DD/MM/YYYY, all text in lowercase).

5. Data Transformation — Convert data into analysis-ready format.

TRANSFORM

"Transform" is the core step of taking messy data and putting it in the right shape. It includes:

• Reshaping data — Changing layout: rows to columns or vice versa.

• Combining data — Merging from multiple sources into one dataset.

• Creating new variables — Calculating new features from existing columns (e.g., BMI from height and weight).

• Data Normalization — Scaling values to a common range (0 to 1).

• Applying mathematical functions — Log, square root, etc. to improve data distribution.

MERGE

Combining multiple datasets into one:

• Merging — Combines datasets based on common columns (like SQL JOIN).

• Joining — Same as merging but specifically when DataFrames share indexes.

• Concatenating — Stacks data along an axis (adding rows or columns).

RESHAPE

Reshaping changes how data is organized by moving information between rows and columns. Data comes in two shapes:

• Wide format — One row per subject. Each variable gets its own column. Example: student has columns Math, Science, English.

• Long format — One row per observation. Variables are stacked in one column. Example: student appears 3 times, once per subject.

Wide is easier to read for humans. Long is easier to process with code and models.

Python — Basic Wrangling Steps
import pandas as pd
import numpy as np

data = {'Name': ['Jai', 'Princi', 'Gaurav', 'Anuj', 'Ravi'],
        'Age':  [17, 17, 18, 17, 18],
        'Gender': ['M', 'F', 'M', 'M', 'M'],
        'Marks': [90, 76, 'NaN', 74, 65]}
df = pd.DataFrame(data)

# Step 1: Replace NaN string with column mean
c = avg = 0
for ele in df['Marks']:
    if str(ele).isnumeric():
        c += 1; avg += ele
avg /= c
df = df.replace(to_replace='NaN', value=avg)

# Step 2: Encode gender M=0, F=1
df['Gender'] = df['Gender'].map({'M': 0, 'F': 1}).astype(float)

# Step 3: Filter top scorers
df = df[df['Marks'] >= 75].copy()
df.drop('Age', axis=1, inplace=True)
3. Combining and Merging Datasets
Theory

Pandas provides three main ways to combine data from multiple DataFrames:

• pd.merge() — Connects rows based on one or more key columns. Similar to SQL database JOIN operations. This is the most common way to combine data that shares a common column.

• pd.concat() — Concatenates (stacks) objects along an axis. Used when both DataFrames have the same columns and you want to add more rows, OR when you want to add new columns side by side.

• combine_first() — Patches (fills) missing values in one DataFrame using values from another. Used when two datasets overlap and one has gaps the other can fill.

Types of Joins — pd.merge()

What is a Join?

A join is a way to combine rows from two tables based on a related column (called the key). The type of join determines which rows are kept in the result when keys don't match perfectly.

Join Typehow= valueRows KeptNaN produced?
Inner Join'inner' (default)Only rows where key EXISTS in BOTH tablesNo
Left Join'left'ALL rows from LEFT table + matching from rightYes (where right has no match)
Right Join'right'ALL rows from RIGHT table + matching from leftYes (where left has no match)
Outer Join'outer'ALL rows from BOTH tables combinedYes (wherever either side is missing)
Python — All Merge/Join Types
df1 = pd.DataFrame({'key': ['b','b','a','c','a','a','b'], 'data1': range(7)})
df2 = pd.DataFrame({'key': ['a','b','d'],         'data2': range(3)})

# Inner: only rows where key matches in BOTH df1 and df2
# Result: only 'a' and 'b' rows (not 'c' from df1, not 'd' from df2)
pd.merge(df1, df2, on='key', how='inner')

# Left: keep ALL df1 rows; 'c' gets NaN for data2
pd.merge(df1, df2, on='key', how='left')

# Right: keep ALL df2 rows; 'd' gets NaN for data1
pd.merge(df1, df2, on='key', how='right')

# Outer: keep ALL rows from both; fill missing with NaN
pd.merge(df1, df2, on='key', how='outer')

# Different column names in each df
pd.merge(df3, df4, left_on='lkey', right_on='rkey')

# Merge using the INDEX as the join key
pd.merge(left1, right1, left_on='key', right_index=True)

Concatenation — pd.concat()

Theory

Concatenation means stacking data. Unlike merge (which joins on keys), concat just appends rows or columns without needing a common key.

• axis=0 (default) — Stack rows on top of each other (make it taller).

• axis=1 — Place columns side by side (make it wider).

• join='outer' (default) — Keep all indexes (fill NaN where missing).

• join='inner' — Keep only common indexes (discard others).

• keys=[] — Adds a hierarchical index label so you know which source each row came from.

Python — Concatenation
s1 = pd.Series([0, 1], index=['a', 'b'])
s2 = pd.Series([2, 3], index=['c', 'd'])
s3 = pd.Series([5, 6], index=['f', 'g'])

# Stack rows (default axis=0)
pd.concat([s1, s2, s3])

# Stack as columns side by side
pd.concat([s1, s2, s3], axis=1)

# Only keep common indexes
pd.concat([s1, s2], axis=1, join='inner')

# Add labels to track which source each row came from
pd.concat([s1, s2, s3], keys=['one', 'two', 'three'])

# For DataFrames — ignore original index, create fresh one
pd.concat([df1, df2], ignore_index=True)

# DataFrame with column labels
pd.concat([df1, df2], axis=1, keys=['level1', 'level2'])

Combining with Overlap — combine_first()

Theory

combine_first() fills NaN values in one object using values from another. Think of it as "use my values where I have them; otherwise borrow from you."

For DataFrames, it works column by column — like patching holes in one dataset with data from another.

Python — combine_first
s1 = pd.Series([1, np.nan, 3, np.nan], index=['a','b','c','d'])
s2 = pd.Series([10, 20, 30, 40],   index=['b','c','d','e'])

result = s1.combine_first(s2)
# a=1.0 (from s1), b=10 (s1 was NaN → patched from s2)
# c=3.0 (from s1), d=30 (patched), e=40 (only in s2)
4. Reshaping and Pivoting
Theory

Reshaping changes how data is organized — moving information between rows and columns without changing the actual values.

You need to reshape because certain operations and machine learning models require a specific format. Some tools need wide format; others need long format.

Wide vs Long Format

Wide Format

Each row = one subject. Each variable gets its own column. Easier for humans to read.

Example: Rahul | Math:85 | Science:90 | English:88

Long Format

Each row = one observation. Subject repeats. Easier for computers to process.

Example: Rahul | Math | 85 / Rahul | Science | 90 / Rahul | English | 88

stack() and unstack() — Hierarchical Index Reshaping

Theory

stack() — Rotates (pivots) columns INTO rows. Makes data longer. The column names become part of the row index.

unstack() — Rotates rows BACK into columns. Makes data wider. Used to reverse a stack operation.

Python — stack & unstack
data = pd.DataFrame(np.arange(6).reshape((2, 3)),
    index=pd.Index(['Ohio', 'Colorado'], name='state'),
    columns=pd.Index(['one', 'two', 'three'], name='number'))

result = data.stack()    # Wide DataFrame → Long Series
# state    number
# Ohio     one       0
#          two       1
# Colorado one       3 ...

result.unstack()          # Undo stack → back to wide DataFrame

pivot() — Long to Wide Format

Theory

pivot() transforms long format into wide format. It takes unique values from one column and turns them into new column headers, with corresponding values filling in the cells.

Parameters: index = which column becomes the row identifier, columns = which column's values become new column headers, values = which column has the actual data values.

Python — pivot Long → Wide
df = pd.DataFrame({
    'Student': ['Rahul','Rahul','Rahul','Anita','Anita','Anita'],
    'Subject': ['Math','Science','English','Math','Science','English'],
    'Marks':   [85, 90, 88, 78, 82, 80]
})

wide = df.pivot(index='Student', columns='Subject', values='Marks')
# Output:
# Subject  English  Math  Science
# Anita        80    78       82
# Rahul        88    85       90

melt() — Wide to Long Format (Inverse of pivot)

Theory

melt() does the opposite of pivot. It collapses multiple columns into two: one column holds the old column names (variable), another holds the values.

Parameters: id_vars = columns to keep as identifiers (not melted), var_name = name for the new "variable" column, value_name = name for the new "value" column.

Python — melt Wide → Long
wide = pd.DataFrame({ 'Student': ['Rahul', 'Anita'], 'Math': [85, 78], 'Science': [90, 82], 'English': [88, 80] }) long = wide.melt(id_vars='Student', var_name='Subject', value_name='Marks') # Output: # Student Subject Marks # Rahul Math 85 # Rahul Science 90 ...
5. Data Cleaning and Preparation
Theory

Data cleaning is a critical phase — it takes up 80% or more of a data analyst's time. It involves loading, cleaning, transforming, and rearranging data to make it suitable for analysis or modeling.

Pandas uses the floating-point value NaN (Not a Number) to represent missing data. This is called a sentinel value — a special marker that can be easily detected.

NA Handling Methods

MethodWhat it does
isnull()Returns True for each element that is missing/NaN
notnull()Returns True for each element that is NOT missing (opposite of isnull)
dropna()Filters out/removes rows or columns that have missing values
fillna()Fills in missing values with a specified value or method

dropna() — Remove Missing Values

Theory

dropna() removes rows (or columns) that contain missing values. By default, it drops any row that has even one NaN. You can control this behavior with parameters.

Key parameters: how='any' (default) drops if ANY value is NaN; how='all' only drops if ALL values are NaN; axis=1 drops columns instead of rows; thresh=N keeps rows that have at least N non-NaN values.

Python — dropna
df = pd.DataFrame([[1, 6.5, 3],
                   [1, np.nan, np.nan],
                   [np.nan, np.nan, np.nan],
                   [np.nan, 6.5, 3]])

df.dropna()              # Remove rows with ANY NaN → keeps only row 0
df.dropna(how='all')     # Remove rows where ALL values are NaN → removes row 2
df.dropna(thresh=2)      # Keep rows with at least 2 non-null values
df.dropna(axis=1, how='all')  # Drop COLUMNS where all values are NaN

fillna() — Fill Missing Values

Theory

Instead of removing missing values, you can fill them in with something appropriate. Common strategies:

• Fill with a constant (0, -1) — simple but may distort statistics.

• Fill with mean/median — most statistically sound for numerical data.

• Forward fill (ffill) — copies the previous row's value. Good for time series.

• Backward fill (bfill) — copies the next row's value.

The limit parameter restricts how many consecutive NaNs to fill.

Python — fillna
# Fill all NaN with 0
df.fillna(0)

# Fill each column with a different value (using dict)
df.fillna({1: 0.5, 2: 0})

# Forward fill — use previous row's value
df.fillna(method='ffill')

# Forward fill but max 2 consecutive NaNs
df.fillna(method='ffill', limit=2)

# Fill with column mean (most common in practice)
df.fillna(df.mean())

# Fill a Series with its mean
data = pd.Series([1, np.nan, 3.5, np.nan, 7])
data.fillna(data.mean())   # NaN → 3.833

# Modify in-place (don't create new df)
df.fillna(0, inplace=True)
6. Data Transformation

a) Removing Duplicates

Theory

Duplicate rows appear in datasets because of data entry errors, merging overlapping datasets, or data collection issues. Keeping duplicates inflates counts and skews analysis.

duplicated() returns a boolean Series — True for rows that are exact copies of a previous row. drop_duplicates() removes those rows. By default, both keep the first occurrence and remove the rest.

Python — duplicates
data = pd.DataFrame({'k1': ['one','two']*3 + ['two'],
                      'k2': [1,1,2,3,3,4,4]})

data.duplicated()               # Mark duplicates as True
data.drop_duplicates()           # Remove duplicate rows
data.drop_duplicates(['k1'])     # Deduplicate based on only 'k1'
data.drop_duplicates(keep='last') # Keep last occurrence instead of first

b) Transforming Data Using map()

Theory

map() applies a function or dictionary mapping to every element in a Series. It is used for element-wise transformation — changing or replacing individual values based on a rule or lookup table.

Example use: converting letter grades to numeric values, categorizing marks into grade labels, encoding strings.

Python — map transformation
data = pd.DataFrame({
    'student': ['AMAN', 'Riya', 'aman', 'Sohan'],
    'marks':   [85, 92, 78, 88]
})
# Lowercase all names
data['student'].str.lower()

# Map marks to grades using a function + dict
grade_map = {'a': 'Excellent', 'b': 'Good', 'c': 'Average'}
data['grade'] = data['marks'].map(
    lambda x: grade_map['a' if x >= 90 else 'b' if x >= 75 else 'c']
)

c) Replacing Values

Theory

replace() substitutes one value with another throughout a Series or DataFrame. Common use: replacing sentinel values (like -999 or 99999 which mean "unknown") with NaN so pandas can handle them properly.

Python — replace
data = pd.Series([1, -999, 2, -999, -1000, 3])

data.replace(-999, np.nan)                 # Replace one value
data.replace([-999, -1000], np.nan)       # Replace multiple with same
data.replace({-999: np.nan, -1000: 0})   # Replace each with different value

d) Renaming Axis Indexes

Theory

rename() creates a new DataFrame with renamed row/column labels. You can pass a function (like str.upper) or a dict to rename specific labels. Using data.index = data.index.map(func) modifies the index in-place.

Python — rename
data = pd.DataFrame(np.arange(12).reshape((3,4)), index=['Ohio','Colorado','New York'], columns=['one','two','three','four']) # Apply a function to all index/column labels data.rename(index=str.title, columns=str.upper) # Rename specific labels using a dict data.rename(index={'Ohio': 'Indiana'}, columns={'three': 'peekaboo'}) # Modify index in-place data.index = data.index.map(lambda x: x[:4].upper())

e) Permutation and Random Sampling

Theory

Permutation means randomly reordering the rows of a DataFrame. Random sampling means selecting a random subset of rows. Both are useful in machine learning for shuffling data before splitting into train/test sets.

np.random.permutation(n) produces an array of random integers 0 to n-1. df.take(array) reorders rows according to that array.

Python — permutation
df = pd.DataFrame(np.arange(20).reshape((5, 4)))
sampler = np.random.permutation(5)  # e.g., [3, 1, 4, 2, 0]
df.take(sampler)  # Reorder rows randomly

f) Dummy Variables (One-Hot Encoding)

Theory

Machine learning models work with numbers, not text. Dummy variables (also called one-hot encoding) convert a categorical column (like "color": red/blue/green) into multiple binary columns, one for each unique category. Each column contains 1 if that category applies, 0 otherwise.

If a column has k distinct values, this creates k binary columns.

Python — get_dummies
df = pd.DataFrame({'key': ['b','b','a','c','a','b'], 'data1': range(6)})
dummies = pd.get_dummies(df['key'])
# Output: columns a, b, c — each row has 1 in the matching column, 0 elsewhere
#    a  b  c
# 0  0  1  0
# 1  0  1  0
# 2  1  0  0 ...
7. Binning (Discretization)
Definition

Binning (also called Discretization) is the process of converting continuous numerical data into discrete categories (called bins or buckets).

Example: Instead of storing exact ages like 23, 45, 67, you group them into bins: "Young (0–30)", "Middle (31–60)", "Senior (61+)".

Why Use Binning?

  • Simplifies data — reduces number of distinct values
  • Reduces noise — minor variations within a bin are ignored
  • Easier analysis & visualization — grouped data is easier to understand
  • Useful in machine learning — some algorithms work better with categorical data

Types of Binning

Equal-Width Binning — pd.cut()

Each bin covers the same range. Example: 0–33, 33–66, 66–100. You define the bin edges. Simple but can create uneven group sizes.

Equal-Frequency Binning — pd.qcut()

Each bin has the same number of data points. Bins are sized by quantiles (percentiles). Good when data is skewed.

Python — pd.cut (equal-width)
df = pd.DataFrame({'student':['Aman','Riya','Sohan','Neha','Karan'],
                   'marks': [40, 55, 70, 85, 90]})

bins   = [0, 33, 66, 100]
labels = ['Low', 'Medium', 'High']
df['category'] = pd.cut(df['marks'], bins=bins, labels=labels)
# 40→Medium, 55→Medium, 70→High, 85→High, 90→High

# Key parameters of pd.cut:
# x = input array, bins = edges or number of bins
# labels = category names, right=True means right edge is included
Python — pd.qcut (equal-frequency)
# q=3 means split into 3 equal-sized groups (tertiles) df['category'] = pd.qcut(df['marks'], q=3, labels=['Low','Medium','High']) # q=4 gives quartiles, q=[0,.25,.5,.75,1.] gives custom quantiles
8. Classing and Standardization

Classing (Classification / Categorization)

Theory

Classing involves converting continuous numerical data into discrete groups or classes. It is essentially binning with meaningful labels. Used for grouping ages into life stages, income into brackets, or exam scores into grades.

Key function: pandas.cut()

Python — Classing with pd.cut
df = pd.DataFrame({'age': [10, 25, 45, 60, 80]})
bins   = [0, 20, 40, 60, 100]
labels = ['Child', 'Young Adult', 'Adult', 'Senior']
df['age_group'] = pd.cut(df['age'], bins=bins, labels=labels)
# 10→Child, 25→Young Adult, 45→Adult, 60→Adult, 80→Senior

Standardization (Z-score Normalization)

Theory

Standardization transforms numerical data so each feature has mean = 0 and standard deviation = 1. This is a critical preprocessing step for machine learning.

Why needed? Many ML algorithms (KNN, SVM, Neural Networks, PCA) are sensitive to the scale of features. If one feature is in thousands (salary) and another is 0–1 (percentage), the large-scale feature dominates. Standardization makes all features equally scaled.

Formula: z = (x − mean) / standard deviation

Two methods in Python:

• Manual (pandas) — gives full control; uses sample std (÷N-1 by default).

• StandardScaler (scikit-learn) — preferred for ML pipelines; uses population std (÷N).

This is why the output values differ slightly between the two methods.

Python — Manual Standardization
df = pd.DataFrame({'feature': [10, 20, 30, 40, 50]})

mean = df['feature'].mean()   # 30
std  = df['feature'].std()    # 15.81 (sample std, uses N-1)
df['standardized'] = (df['feature'] - mean) / std
# Output: -1.26, -0.63, 0.0, 0.63, 1.26
Python — StandardScaler (Sklearn)
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
df['std_sklearn'] = scaler.fit_transform(df[['feature']])
# Output: -1.41, -0.71, 0.0, 0.71, 1.41
# (slightly different because sklearn uses population std ÷N)
9. Outliers, Noise & Anomalies
Overview

These are three related but different concepts:

• Outliers — extreme values that are far from the rest of the data.

• Noise — random error or disturbance in data that reduces quality.

• Anomalies — unusual patterns or behaviors that don't follow expected rules.

1. Outliers

Definition

An outlier is a data point that is significantly different from other observations. It lies far outside the typical range.

Example: Marks = [85, 90, 88, 92, 300] — 300 is clearly an outlier.

Causes of Outliers
  • Data entry errors (human typo)
  • Measurement error (faulty instrument)
  • Rare but valid cases (genuine extreme)
Detection Methods
  • IQR method — using quartiles
  • Z-score — using standard deviation
  • Boxplot — visual inspection
IQR Method — How It Works

1. Calculate Q1 (25th percentile) and Q3 (75th percentile).

2. IQR = Q3 − Q1 (the middle 50% spread).

3. Lower bound = Q1 − 1.5 × IQR

4. Upper bound = Q3 + 1.5 × IQR

5. Any value below lower bound or above upper bound is an outlier.

Python — IQR Outlier Detection
df = pd.DataFrame({
    'student': ['Aman','Riya','Sohan','Neha','Karan','Pooja'],
    'marks':   [85, 90, 88, 300, 92, 40]  # 300 and 40 are outliers
})

Q1 = df['marks'].quantile(0.25)
Q3 = df['marks'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

outliers  = df[(df['marks'] < lower) | (df['marks'] > upper)]
clean     = df[(df['marks'] >= lower) & (df['marks'] <= upper)]
Python — Z-score Outlier Detection
from scipy import stats

df['zscore'] = stats.zscore(df['marks'])
# Z-score > 3 or < -3 means outlier (3 standard deviations away)
outliers = df[df['zscore'].abs() > 3]

2. Noise

Definition

Noise is random error or unwanted disturbance in data that does not reflect the true underlying pattern. Unlike outliers (which are extreme), noise can be small and scattered throughout the data.

Examples: typo in marks (89 → 98), sensor reading errors, measurement instrument fluctuations.

Characteristics
  • Random and unpredictable
  • Reduces data quality and model accuracy
  • Can't always be distinguished from signal
Handling Methods
  • Smoothing (moving average)
  • Replace with median
  • Binning (smooth by groups)
  • Filtering
Python — Moving Average (Noise Smoothing)
df = pd.DataFrame({'marks': [85, 90, 200, 88, 87, 300, 86]})

# Method 1: Rolling average (window=3 means average of 3 consecutive values)
df['smoothed'] = df['marks'].rolling(window=3).mean()

# Method 2: Replace extreme values with median
median = df['marks'].median()
df['clean'] = df['marks'].apply(lambda x: median if x > 150 else x)

# Method 3: Binning — smooth by group mean
df['bin'] = pd.cut(df['marks'], bins=3)
df['smooth_bin'] = df.groupby('bin')['marks'].transform('mean')

3. Anomalies

Definition

Anomalies are unusual patterns or rare observations that do not follow expected behavior. Unlike simple outliers (single extreme values), anomalies can be contextual (normal in one context, abnormal in another).

Examples: A student who always scores 90+ suddenly scoring 15 in one test. A credit card transaction of $5,000 from someone who never spends more than $200 (fraud detection).

TypeDefinitionExample
Point AnomalyA single data point that is far from the restOne transaction of $10,000 in a mostly $50 account
Contextual AnomalyData point that is abnormal in a specific context but normal elsewhere30°C temperature in December (winter) but normal in June
Collective AnomalyA group of related data points that are anomalous together5 consecutive failed logins from same IP (might be normal individually)
Python — Detect Anomaly
df = pd.DataFrame({
    'student': ['Aman'] * 5,
    'test':    ['T1','T2','T3','T4','T5'],
    'marks':   [88, 90, 92, 15, 91]  # 15 is anomaly — sudden drop
})
anomaly = df[df['marks'] < 40]   # Simple threshold-based detection

11. String Manipulation
Theory

String manipulation is essential for cleaning text data — fixing names, extracting emails, standardizing formats. Python provides three levels of string tools:

1. Built-in string methods — basic operations on individual strings.

2. Regular Expressions (re module) — complex pattern matching in text.

3. Pandas vectorized str methods — apply string operations to entire columns at once, safely skipping NaN values.

Built-in String Methods

Regular Expressions (re module)

Theory

Regular expressions (regex) are a way to describe patterns in text using a special syntax. Python's built-in re module applies these patterns to strings.

Three main categories of regex functions: pattern matching (find), substitution (replace), and splitting.

Common pattern symbols: \s+ = one or more whitespace, \d = digit, [A-Z] = any capital letter, + = one or more, {2,4} = 2 to 4 times.

FunctionWhat it does
re.split(pattern, text)Split text wherever pattern matches
re.findall(pattern, text)Return a list of all matches
re.search(pattern, text)Find first match anywhere in the string
re.match(pattern, text)Match only at the START of the string
re.compile(pattern)Pre-compile a pattern for reuse (faster)
regex.sub(replacement, text)Replace all matches with replacement
Python — Regex examples
import re

# Split on whitespace
re.split(r'\s+', "foo bar\t baz")    # → ['foo','bar','baz']

# Find all email addresses in text
text = """Dave dave@google.com
Steve steve@gmail.com"""
pattern = r'[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}'
regex = re.compile(pattern, flags=re.IGNORECASE)
regex.findall(text)   # → ['dave@google.com', 'steve@gmail.com']

# search() finds first match anywhere; match() only checks start
m = regex.search(text)   # returns match object
print(regex.match(text))  # None (no email at very start)

Vectorized String Functions in Pandas

Theory

When you have a column of strings in a DataFrame, you can't apply regular Python string methods directly because the column might have NaN values (which would crash). Pandas provides a str accessor (Series.str) that applies string methods to an entire column while automatically skipping NaN values.

Python — pandas str accessor

                  'Rob':  'rob@gmail.com',
                  'Wes':  np.nan})

data.str.lower()               # all lowercase
data.str[:5]                   # first 5 chars (Dave→dave@)
data.str.split('@')            # split on @
data.str.replace('gmail', 'yahoo')

# Regex in str methods

12. Introduction to Matplotlib
Definition

Matplotlib is a comprehensive Python library for creating static, animated, and interactive visualizations. It was created by John D. Hunter in 2003 and is the most widely used plotting library in Python.

The pyplot module (matplotlib.pyplot) provides a MATLAB-like interface for creating figures and plots. It manages the figure, axes, and plot elements automatically.

Key Features

  • Supports a wide variety of plots: line, bar, scatter, histogram, pie, box, etc.
  • Highly customizable: colors, labels, titles, ticks, legends, annotations
  • Supports LaTeX for mathematical expressions in text (like $y = \sin(x)$)
  • Can export to many formats: PNG, PDF, SVG, EPS
  • Integrates well with NumPy, Pandas, and SciPy

13. Making Subplots
Theory

Subplots allow you to display multiple plots in a single figure arranged in a grid. This is useful for comparing different datasets, different variables, or different visualizations side by side.

Two main approaches:

• plt.subplot(rows, cols, index) — simple but limited; activates one subplot at a time.

• plt.subplots(rows, cols) — returns a Figure and an array of Axes objects, giving more control. This is the recommended approach.

Python — plt.subplots() (recommended)
fig, axes = plt.subplots(2, 2, figsize=(10, 8))

axes[1, 0].plot(x, np.tan(x));  axes[1, 0].set_title('Tangent')
axes[1, 1].plot(x, np.exp(x));  axes[1, 1].set_title('Exponential')

plt.tight_layout()   # Auto-fix overlapping labels/titles
plt.show()

14. Controlling Axes, Ticks, Labels & Legends

Controlling Axes

Theory

Axes control defines what range of data is visible in the plot, how the scale works (linear vs logarithmic), and how gridlines appear.

xlim() and ylim() set the visible data range. xscale('log') changes to logarithmic scale (useful for data spanning many orders of magnitude). grid(True) adds gridlines for easier reading.

Theory

Ticks are the small markers on axes. Tick labels are the text next to each tick. You can customize their position, content, size, rotation, and color using xticks() and tick_params().

Python — Custom Ticks

Legends

Theory

A legend identifies different data series in a plot. To use a legend: (1) add a label= parameter to each plot() call, then (2) call plt.legend() to display it. The loc parameter controls position.

Python — Legends

plt.legend(loc='best', fontsize=10) # Best = auto-find best spot

Annotations

Theory

Annotations add text and arrows to highlight specific points or regions in a plot. Two main functions:

• plt.text(x, y, "text") — places text at coordinates (x, y).

• plt.annotate("text", xy=point, xytext=text_pos, arrowprops=...) — adds text with an optional arrow pointing to a specific data point. xy is where the arrow points, xytext is where the text label sits.

Python — Annotations

Theory — What & When

A scatter plot displays the relationship between two numerical variables by placing dots on a Cartesian plane. Each dot is one data point (x, y).

When to use: To study correlation, identify clusters, detect outliers, visualize bivariate distributions.

What it shows: Whether two variables have a positive relationship (both increase), negative relationship (one increases as other decreases), or no relationship.

Theory — What & When

A line plot connects data points with a continuous line. It shows how a value changes over a sequence or time.

When to use: Time series data (stock prices, temperatures over months), trends over ordered categories, comparing multiple trends.

Theory — What & When

A bar plot represents categorical data with rectangular bars. The length (or height) of each bar is proportional to the value it represents.

When to use: Comparing quantities across different categories (Math vs Science scores), showing frequency distributions, displaying ranked data.

Histogram

Theory — What & When

A histogram shows the frequency distribution of numerical data. It groups data into ranges called bins and shows how many data points fall into each bin.

When to use: Understand the spread and shape of data, check if data is normally distributed, find skewness (data bunching left/right), identify central tendency.

Bins — the number of bars. More bins = more detail, more noise. Fewer bins = smoother, less detail. The KDE curve (Kernel Density Estimate) adds a smooth distribution line.

Key difference from bar plot: Bar plot is for categories. Histogram is for continuous numerical data.

Python — Histogram
# Matplotlib

plt.title('Distribution'); plt.show()

# Seaborn — adds KDE curve automatically with kde=True
sns.histplot(data, bins=30, kde=True, color='blue')
plt.title('Histogram with KDE'); plt.show()

17. 3D Plots of Surfaces
Theory

Matplotlib's mpl_toolkits.mplot3d module enables creation of 3D visualizations. 3D plots are used to visualize mathematical functions of two variables, terrain data, or any data with three dimensions.

To create a 3D plot, you need to create a 3D axes object using fig.add_subplot(111, projection='3d').

For surface plots, you use np.meshgrid(x, y) to create a 2D grid of X and Y coordinates, then compute Z values for each (x,y) pair.

Python — 3D Surface Plot
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D
import numpy as np

# Create the x,y grid
x = np.linspace(-5, 5, 100)
y = np.linspace(-5, 5, 100)
X, Y = np.meshgrid(x, y)        # 2D grid of coordinates
Z = np.sin(np.sqrt(X**2 + Y**2)) # Z = f(x, y)

fig = plt.figure(figsize=(10, 7))
ax = fig.add_subplot(111, projection='3d')

surf = ax.plot_surface(X, Y, Z, cmap='viridis', alpha=0.9)
fig.colorbar(surf, ax=ax, shrink=0.5, label='Z value')
ax.set_xlabel('X'); ax.set_ylabel('Y'); ax.set_zlabel('Z')
ax.set_title('3D Surface Plot')
ax.view_init(elev=30, azim=60)   # Camera angle
plt.show()
